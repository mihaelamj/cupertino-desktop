# Mobile Layouts and Components Adaptivity

This directory contains layout and UI component guidelines for Cupertino mobile implementations, ensuring compliance with the Apple Human Interface Guidelines (HIG).

## Contents

- [mobile-orientation-adaptivity-matrix.md](mobile-orientation-adaptivity-matrix.md) - **Mobile Orientation and Size Class Adaptivity Matrix**: Specifies size classes and navigation layout behaviors for standard and Max iPhones, full screen iPads, iPad multitasking states (Slide Over and Split View), and macOS.
- [mobile-ui-components-reference.md](mobile-ui-components-reference.md) - **Mobile UI Components Behavior and Styling**: Details component-specific visual states, headers, actions, and transition behaviors for the core UI controls (databases list, frameworks list, search field, document reader, and sort controls).
- [actions-taken.md](actions-taken.md) - **Actions Taken Log**: A historical record detailing the research, refactoring, UI automation test runs, and documentation creation performed to satisfy these design requirements.
